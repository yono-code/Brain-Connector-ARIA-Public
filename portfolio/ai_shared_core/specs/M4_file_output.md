# [M4] GUI → ファイル出力 — スペックキット

## 概要

- **目的**: GUI操作後、2秒以内に `.ai-context/` 以下の全ファイルを自動更新する
- **ゴール**: タスク作成や状態変更が `status.md` / `architecture.mermaid` / ADR に反映される
- **前提条件**: M1-3（ファイル書き込みモジュール）、M3-8（Store→Extension Host 橋渡し）完了
- **推定工数**: 2週間

## 技術仕様・実装ガイド

### M4-1 + M4-8: Debounce タイマー + 自己ループ防止（Extension Host 側）

`src/extension.ts`（メインエントリーポイント）での処理:

```typescript
import * as vscode from 'vscode';
import { AriaPanel } from './extension/aria-panel';
import { AriaStateWatcher } from './extension/aria-state-watcher';
import { writeAllAiContextFiles } from './extension/file-writer';
import { loadAriaState, DEFAULT_ARIA_STATE } from './extension/persistence';
import { requireWorkspace } from './extension/utils/workspace';
import { WebviewToExtensionMessage, AriaState } from './shared/types';

export function activate(context: vscode.ExtensionContext) {
  let currentState: AriaState = { ...DEFAULT_ARIA_STATE };
  let debounceTimer: ReturnType<typeof setTimeout> | undefined;
  let watcher: AriaStateWatcher | undefined;

  const openPanel = async () => {
    const workspacePath = requireWorkspace();
    if (!workspacePath) return;

    // 保存済みの状態を読み込む
    currentState = await loadAriaState(workspacePath);

    // FileSystemWatcher を開始
    if (!watcher) {
      watcher = new AriaStateWatcher(
        workspacePath,
        async (rawJson) => {
          // 外部変更を受け取ったら Webview に通知
          try {
            const parsed = JSON.parse(rawJson) as AriaState;
            currentState = parsed;
            panel.postMessage({ type: 'STATE_UPDATED', payload: currentState });
          } catch {
            // パースエラーは無視（GUIはクラッシュしない）
          }
        }
      );
      watcher.start();
      context.subscriptions.push({ dispose: () => watcher?.dispose() });
    }

    // Webview パネルを開く
    const panel = AriaPanel.createOrShow(
      context.extensionUri,
      async (message: WebviewToExtensionMessage) => {
        if (message.type === 'READY') {
          // Webview 準備完了 → 現在の状態を送信
          panel.postMessage({ type: 'INIT_STATE', payload: currentState });
        }

        if (message.type === 'STATE_CHANGED') {
          // Webview からの状態変更 → Debounce してファイル書き込み
          currentState = message.payload;

          if (debounceTimer) clearTimeout(debounceTimer);
          debounceTimer = setTimeout(async () => {
            // 書き込み開始マーク（Watcher の誤検知防止）
            watcher?.markWriteStart();

            await writeAllAiContextFiles(
              workspacePath,
              currentState,
              () => watcher?.markWriteStart() ?? undefined
            );
          }, 2000); // 2秒 Debounce
        }
      }
    );
  };

  context.subscriptions.push(
    vscode.commands.registerCommand('aria.openPanel', openPanel)
  );
}

export function deactivate() {}
```

### M4-2: status.md 生成ロジック【最重要】

**`src/extension/generators/status-md-generator.ts`**:

```typescript
import { KanbanTask, TaskStatus } from '../../shared/types';

// status.md の先頭に付与するガードヘッダー（第1の壁）
const GUARD_HEADER = `<!--
  ============================================================
  ARIA STATUS FILE — DO NOT MODIFY THIS HEADER BLOCK
  ============================================================
  This file is AUTO-GENERATED by the ARIA VS Code Extension.

  RULES FOR AI AGENTS:
  1. DO NOT remove or modify task ID anchors (<!-- task-id-... -->)
  2. DO NOT change section headers (## Inbox, ## Todo, etc.)
  3. DO NOT reorder tasks within sections
  4. You MAY change checkbox state: [ ] → [x] or [x] → [ ]
  5. You MAY change task titles (the text after the checkbox)
  6. You MAY add new tasks WITHOUT an ID anchor (they will be
     auto-assigned to Inbox for human review)

  FAILURE TO FOLLOW THESE RULES WILL CORRUPT THE STATE.
  ============================================================
-->

`;

const STATUS_ICON: Record<TaskStatus, string> = {
  'Inbox':       '📥',
  'Todo':        '⬜',
  'In Progress': '🔵',
  'Done':        '✅',
};

const SECTION_ORDER: TaskStatus[] = ['Inbox', 'Todo', 'In Progress', 'Done'];

export function generateStatusMd(tasks: Record<string, KanbanTask>): string {
  const taskList = Object.values(tasks);

  // セクションごとにタスクを分類
  const sections: Record<TaskStatus, KanbanTask[]> = {
    'Inbox': [],
    'Todo': [],
    'In Progress': [],
    'Done': [],
  };

  for (const task of taskList) {
    sections[task.status].push(task);
  }

  const lines: string[] = [GUARD_HEADER];

  for (const status of SECTION_ORDER) {
    const icon = STATUS_ICON[status];
    lines.push(`## ${icon} ${status}\n`);

    const sectionTasks = sections[status];
    if (sectionTasks.length === 0) {
      lines.push('_（タスクなし）_\n');
    } else {
      for (const task of sectionTasks) {
        const checkbox = task.status === 'Done' ? '[x]' : '[ ]';
        // IDアンカーを行末に付与（不可視のHTMLコメント）
        lines.push(`- ${checkbox} ${task.title} <!-- task-id-${task.id} -->`);
      }
      lines.push('');
    }
  }

  return lines.join('\n');
}
```

import * as vscode from 'vscode';
import * as path from 'path';
import * as fs from 'fs/promises';

const CURSORRULES_CONTENT = `# ARIA Project Rules for AI Assistants
# Auto-generated by ARIA VS Code Extension — DO NOT DELETE

## CRITICAL: .ai-context/ Directory Rules

### status.md
- NEVER remove task ID anchors: <!-- task-id-task-XXXXXXXX -->
- NEVER modify section headers: ## 📥 Inbox, ## ⬜ Todo, etc.
- You MAY change [ ] to [x] to mark tasks as done
- You MAY add new tasks WITHOUT IDs (they will go to Inbox)

### aria-state.json
- This is the MASTER STATE. Handle with care.
- DO NOT change task IDs
- DO NOT add fields that don't exist in the schema
- Valid status values: "Todo", "In Progress", "Done", "Inbox"

### architecture.mermaid
- This is READ-ONLY for AI. Do not modify.
- It is auto-generated from the GUI canvas.

## Development Philosophy
- Quality over Speed
- Single Source of Truth: aria-state.json
- All ambiguities → ask the human developer first
`;

// プロジェクトルートに .cursorrules を生成する（既存があれば上書きしない）
export async function generateCursorRulesIfNotExists(
  workspacePath: string
): Promise<void> {
  const filePath = path.join(workspacePath, '.cursorrules');

  try {
    await fs.access(filePath);
    // ファイルが存在する → 何もしない
  } catch {
    // ファイルが存在しない → 生成する
    await fs.writeFile(filePath, CURSORRULES_CONTENT, 'utf-8');
    vscode.window.showInformationMessage(
      'ARIA: .cursorrules を生成しました。'
    );
  }
}
```

### M4-5: architecture.mermaid 生成（C4 Context レベル）

**`src/extension/generators/architecture-generator.ts`**:

```typescript
import { AriaNode, AriaEdge } from '../../shared/types';

export function generateArchitectureMermaid(
  nodes: AriaNode[],
  edges: AriaEdge[]
): string {
  const lines: string[] = [
    '```mermaid',
    'C4Context',
    '  title ARIA Project — System Architecture (C4 Context Level)',
    '',
  ];

  // ノードを Mermaid の C4 記法に変換
  for (const node of nodes) {
    const label = node.data.label.replace(/"/g, '\\"');
    const description = (node.data.description ?? '').replace(/"/g, '\\"');
    const technology = node.data.technology ? ` [${node.data.technology}]` : '';

    if (node.type === 'c4-container' || node.type === 'c4-component') {
      lines.push(
        `  Container(${node.id}, "${label}", "${technology}", "${description}")`
      );
    } else if (node.type === 'mindmap') {
      lines.push(
        `  System(${node.id}, "${label}", "${description}")`
      );
    }
  }

  lines.push('');

  // エッジを関係として変換
  for (const edge of edges) {
    const label = (edge.label ?? '').replace(/"/g, '\\"');
    lines.push(`  Rel(${edge.source}, ${edge.target}, "${label}")`);
  }

  lines.push('```');

  return lines.join('\n');
}
```

### M4-6: ADR Markdown 生成

**`src/extension/generators/adr-generator.ts`**:

```typescript
import * as vscode from 'vscode';
import * as path from 'path';
import { ADR } from '../../shared/types';

export async function generateAdrFiles(
  aiContextPath: string,
  adrs: Record<string, ADR>
): Promise<void> {
  const encoder = new TextEncoder();
  const adrDir = path.join(aiContextPath, 'adr');
  let sequence = 1;

  for (const adr of Object.values(adrs)) {
    const paddedSeq = String(sequence).padStart(4, '0');
    const slug = adr.title
      .toLowerCase()
      .replace(/[^a-z0-9]+/g, '-')
      .replace(/^-|-$/g, '')
      .slice(0, 50);
    const filename = `ADR-${paddedSeq}-${slug}.md`;
    const filePath = vscode.Uri.file(path.join(adrDir, filename));

    const content = formatAdr(sequence, adr);
    await vscode.workspace.fs.writeFile(filePath, encoder.encode(content));

    sequence++;
  }
}

function formatAdr(seq: number, adr: ADR): string {
  const rejectedList = adr.rejectedOptions.length > 0
    ? adr.rejectedOptions.map((opt) => `- ${opt}`).join('\n')
    : '- （なし）';

  return `# ADR-${String(seq).padStart(4, '0')}: ${adr.title}

**作成日**: ${adr.createdAt.split('T')[0]}
**ステータス**: 決定済み
**関連ノード**: ${adr.linkedNodeId}

## 決定内容

${adr.decision || '（未記載）'}

## 検討した代替案（採用しなかった理由）

${rejectedList}

---
*このファイルはARIA VS Code Extensionにより自動生成されます。*
*変更はGUIのADR編集パネルから行ってください。*
`;
}
```

## ハマりどころ

### ハマり1: 自己書き込みループ（最重要）

`writeAllAiContextFiles` → FileSystemWatcher 発火 → `onExternalChange` → `STATE_UPDATED` → Webview → `STATE_CHANGED` → `writeAllAiContextFiles` → ...

**必ず** `watcher.markWriteStart()` を `writeAllAiContextFiles` の前に呼ぶこと。
M1-4 の `AriaStateWatcher.markWriteStart()` を参照。

### ハマり2: Debounce の二重設定

- Webview 側（Zustand Store）: 300ms デバウンス（UI操作のバーストを防ぐ）
- Extension Host 側: 2秒 デバウンス（ファイル書き込みトリガー）

この二段階デバウンスは意図的な設計。混同しないこと。

### ハマり3: status.md の改行コード

Windows 環境では改行が `\r\n` になる場合がある。`writeFile` に渡す前に `\n` に統一すること（パーサー（M5）で CRLF/LF 両対応するが、出力は LF 統一を推奨）。

## 完了条件（Acceptance Criteria）

1. タスクを1つ追加すると、2〜3秒後に `.ai-context/status.md` が自動生成される
2. `status.md` に各タスクの IDアンカー `<!-- task-id-task-xxxxxxxx -->` が付与されている
3. `status.md` の先頭にガードヘッダーが含まれている
4. ノードを追加すると `architecture.mermaid` が更新される
5. ADRを追加すると `adr/ADR-0001-xxxx.md` が生成される
6. 初回起動時に `.cursorrules` がプロジェクトルートに生成される（2回目は上書きされない）
7. GUIで10回連続操作しても無限ループが発生しない（ファイルが際限なく書き換わらない）

### 検証手順

```
1. ARIA パネルを開く
2. タスクを3つ追加する
3. .ai-context/status.md を開く → 3タスクがIDアンカー付きで記載されていることを確認
4. タスクのステータスを「Done」に変更する
5. status.md を再確認 → [x] チェックボックスと Done セクションへの移動を確認
6. Activity Monitor / タスクマネージャでファイル書き込みが暴走していないことを確認
```
